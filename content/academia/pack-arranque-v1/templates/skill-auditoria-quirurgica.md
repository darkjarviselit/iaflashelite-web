# Skill — Auditoría quirúrgica

## Para qué sirve

Revisa una idea, página, producto, prompt o decisión con mirada crítica. Ayuda a
detectar contradicciones antes de invertir más tiempo.

## Cuándo usarla

Úsala antes de publicar una landing, vender un producto, automatizar un flujo,
pedir implementación técnica o tomar una decisión importante.

## Cómo copiarla

Pega el prompt y añade el material que quieres revisar. Si tienes Contexto
maestro del proyecto, inclúyelo.

## Prompt

```text
Actúa como auditor quirúrgico de producto, UX, copy y ejecución.

Objetivo:
Quiero que revises el material que te paso y detectes problemas reales antes de
avanzar.

Reglas:
- No reescribas todo si no hace falta.
- Prioriza riesgos P0.
- Distingue hechos, hipótesis y recomendaciones.
- No suavices problemas importantes.
- No inventes datos.
- No propongas cambios fuera del alcance si no son necesarios.

Evalúa:
1. Claridad del objetivo.
2. Coherencia del mensaje.
3. Promesas peligrosas o ambiguas.
4. Riesgos técnicos.
5. Riesgos legales o de confianza.
6. Riesgos UX/CRO.
7. Qué falta para que sea publicable o usable.

Formato de salida:
- Resumen brutal.
- P0: problemas que bloquean.
- P1: mejoras importantes.
- P2: mejoras futuras.
- Recomendación final.
- Siguiente acción concreta.
```

## Ejemplo de uso

```text
Revisa este bloque de venta antes de publicarlo. Quiero saber si promete más de
lo que realmente entrega y qué cambiarías en P0.
```

## Ejemplo completado

```markdown
## Resumen brutal

La idea se entiende, pero el copy promete demasiado y no explica la entrega.

## P0

- La frase "automatiza todo tu negocio" es demasiado amplia.
- Falta decir qué recibe exactamente el comprador.
- No aparece ningún límite ni condición de uso.

## Recomendación

Cambiar la promesa por una entrega concreta y añadir una sección "qué no hace".
```
