## 1. Quiénes somos y por qué usamos este flujo

IAFlashElite construye productos digitales, agentes, automatizaciones y sistemas
de trabajo con IA. No usamos la IA como un chat suelto donde cada conversación
empieza desde cero.

Cuando un proyecto importa, preparamos contexto, instrucciones, límites,
prompts técnicos y revisión. Eso permite trabajar con más control.

Este pack existe para enseñarte ese flujo.

No vas a montar una "IA mágica". Vas a montar proyectos IA con función clara:

- un proyecto para pensar y decidir;
- un proyecto para vender y comunicar;
- un proyecto para crear prompts técnicos;
- un Prompt 0 para arrancar trabajos con Claude Code/Codex;
- un flujo candado para revisar antes de avanzar.

Tu papel no es mirar cómo la IA trabaja sola. Tu papel es dirigir.

> Tú eres el director de orquesta. La IA trabaja, pero tú lees, decides y
> mantienes el control.

## 2. Qué vas a conseguir

Al terminar este pack tendrás:

- un proyecto IA de estrategia y mejoras;
- un proyecto IA de marketing y ventas;
- un proyecto IA prompt builder técnico;
- un Prompt 0 para iniciar cualquier trabajo técnico;
- un flujo candado para trabajar con Claude Code/Codex sin entregar el control;
- una checklist de revisión;
- una plantilla de informe final.

El objetivo no es llenar carpetas. El objetivo es que puedas pasar de:

> "Le pregunto cosas a la IA cuando se me ocurre algo"

> "Tengo un sistema para investigar, planificar, pedir ejecución técnica,
> revisar y decidir el siguiente paso".

## 3. Si vienes del Pack Arranque IA

Si ya hiciste Pack Arranque IA, tienes una base excelente.

Usa estos tres documentos:

1. Currículum IA personal.
2. Contexto Maestro del Proyecto.
3. Instrucciones de Trabajo del Proyecto IA.

No vuelvas a responderlo todo desde cero. Copia esa base en tus nuevos proyectos
IA y úsala como contexto inicial.

Cómo usar cada documento:

- Currículum IA personal: ayuda a que la IA entienda quién eres, cómo trabajas,
  qué sabes hacer y qué límites tienes.
- Contexto Maestro del Proyecto: ayuda a que la IA entienda qué estás
  construyendo, para quién, con qué riesgos y con qué objetivos.
- Instrucciones de Trabajo: ayudan a que la IA responda con el tono,
  profundidad y formato que necesitas.

Si esos documentos están incompletos, actualízalos antes de montar el sistema.
Un proyecto IA es tan útil como el contexto que le das.

## 4. Si empiezas directamente

Puedes usar este pack sin haber hecho Pack Arranque IA.

Pero no empieces con una conversación vacía. Antes de crear los proyectos,
rellena la plantilla:

`plantillas/contexto-minimo-rapido.md`

Ese contexto mínimo debe responder:

- quién eres o qué rol tienes;
- qué proyecto quieres trabajar;
- qué problema resuelve;
- a quién sirve;
- qué existe ya;
- qué no debe prometer;
- qué quieres conseguir en 30 días;
- qué herramientas usas;
- qué límites no se pueden cruzar.

No será tan profundo como Pack Arranque IA, pero te dará una base suficiente
para trabajar con orden.

## 5. Qué es un proyecto IA profesional

Un proyecto IA profesional es un espacio de trabajo con contexto, instrucciones
y función clara.

Puede ser:

- un ChatGPT Project;
- un Claude Project;
- un Gemini Gem;
- un GPT personalizado;
- un espacio equivalente en la herramienta de IA que uses.

No es memoria perfecta. No significa que la IA recuerde todo para siempre ni que
entienda el proyecto como una persona.

Significa algo más práctico:

- mantiene contexto de forma más estable que un chat suelto;
- permite guardar instrucciones;
- puede trabajar con archivos o documentos de referencia si la herramienta lo
  permite;
- reduce la repetición;
- separa tipos de trabajo.

Esa separación es importante.

Si usas el mismo chat para estrategia, copy, prompts técnicos, revisión y dudas
sueltas, el sistema se ensucia. La IA mezcla objetivos. Tú pierdes claridad.

Por eso recomendamos tres proyectos.

## 6. Proyecto 1 — Estrategia y mejoras

### Para qué sirve

Este proyecto es tu mesa de pensamiento.

Úsalo para:

- analizar ideas;
- decidir prioridades;
- detectar riesgos;
- encontrar oportunidades;
- ordenar qué hacer primero;
- decidir qué no construir todavía.

No lo uses para pedir código. No lo uses para resolver todo en una respuesta.
Úsalo para pensar mejor.

### Prompt completo para configurar el proyecto

Copia este prompt en las instrucciones del proyecto o en el primer mensaje de
configuración.

```text
Actúa como mi asesor estratégico de proyectos IA.

Tu función es ayudarme a pensar, priorizar y decidir con claridad. No quiero
respuestas vagas ni motivacionales. Quiero análisis práctico, directo y útil.

Contexto base:
[Pega aquí tu Currículum IA personal o contexto mínimo]
[Pega aquí tu Contexto Maestro del Proyecto]
[Pega aquí tus Instrucciones de Trabajo si las tienes]

Objetivo de este proyecto:
Ayudarme a decidir qué mejorar, qué construir, qué priorizar, qué evitar y qué
riesgos debo revisar antes de avanzar.

Cómo debes trabajar:
- Separa hechos, hipótesis, riesgos y recomendaciones.
- Si falta información importante, pregunta antes de asumir.
- Cuando propongas un plan, divídelo en P0, P1 y P2.
- P0 significa imprescindible para que funcione o para evitar riesgo.
- P1 significa mejora importante.
- P2 significa futuro o sofisticación.
- No prometas resultados económicos.
- No conviertas cualquier idea en buena idea.
- Si algo suena débil, dilo con claridad y propón una alternativa.
- No decidas por mí. Ayúdame a decidir.

Formato preferido:
1. Diagnóstico breve.
2. Opciones posibles.
3. Riesgos.
4. Recomendación.
5. Próximo paso concreto.

Regla principal:
Antes de construir, pensar. Antes de prometer, validar. Antes de avanzar,
definir qué no vamos a hacer.
```

### Primera prueba recomendada

Pídele:

```text
Analiza esta idea y dime si merece una fase P0:
[describe tu idea]

Quiero que separes hechos, hipótesis, riesgos, recomendación y próximo paso.
```

Si la respuesta te ayuda a decidir mejor, el proyecto está funcionando.

## 7. Proyecto 2 — Marketing y ventas

### Para qué sirve

Este proyecto convierte ideas en mensajes claros.

Úsalo para:

- definir ofertas;
- revisar precios;
- escribir copy;
- preparar contenido;
- analizar cliente;
- mejorar posicionamiento;
- detectar promesas peligrosas.

No lo uses para crear humo. El objetivo no es sonar grande. El objetivo es
vender con claridad sin prometer lo que no puedes entregar.

### Prompt completo para configurar el proyecto

```text
Actúa como mi estratega de marketing y ventas con criterio premium.

Tu función es ayudarme a explicar, posicionar y vender proyectos o productos
sin humo, sin exageraciones y sin promesas peligrosas.

Contexto base:
[Pega aquí tu Currículum IA personal o contexto mínimo]
[Pega aquí tu Contexto Maestro del Proyecto]
[Pega aquí tus Instrucciones de Trabajo si las tienes]

Objetivo de este proyecto:
Ayudarme con oferta, precios, copy, contenido, mensajes, posicionamiento y
análisis de cliente.

Cómo debes trabajar:
- Identifica qué se vende realmente.
- Distingue entre beneficio, característica y prueba.
- Detecta promesas exageradas o legalmente arriesgadas.
- No uses urgencia falsa.
- No inventes testimonios, cifras, logos ni autoridad.
- No prometas resultados económicos.
- Si una frase suena genérica, propón una versión más concreta.
- Si una oferta no está clara, ayúdame a simplificarla.

Formato preferido:
1. Lectura del problema.
2. Cliente ideal.
3. Promesa segura.
4. Oferta o mensaje recomendado.
5. Frases a evitar.
6. Próximo paso.

Regla principal:
El copy debe sonar claro, humano y profesional. Mejor una promesa pequeña y
cumplible que una promesa grande que no se sostiene.
```

### Primera prueba recomendada

```text
Revisa esta oferta y conviértela en una versión más clara, concreta y segura:
[pega tu oferta]

No quiero promesas exageradas ni urgencia falsa.
```

## 8. Proyecto 3 — Prompt Builder técnico

### Para qué sirve

Este proyecto prepara prompts para herramientas técnicas como Claude Code,
Codex u otros agentes de programación.

No sustituye a esas herramientas. Las dirige mejor.

Úsalo para definir:

- objetivo;
- contexto;
- archivos a revisar;
- archivos prohibidos;
- límites;
- validaciones;
- formato final;
- criterios de parada.

### Prompt completo para configurar el proyecto

```text
Actúa como mi Prompt Builder técnico para Claude Code, Codex y herramientas de
implementación.

Tu función es convertir objetivos técnicos o de producto en prompts claros,
seguros y ejecutables. No implementas directamente. Preparas el encargo para
otra herramienta y después ayudas a revisar la respuesta.

Contexto base:
[Pega aquí tu Currículum IA personal o contexto mínimo]
[Pega aquí tu Contexto Maestro del Proyecto]
[Pega aquí tus Instrucciones de Trabajo si las tienes]

Objetivo de este proyecto:
Crear prompts de trabajo con límites claros para implementar, auditar,
documentar o revisar proyectos técnicos.

Cada prompt que prepares debe incluir:
- rol solicitado;
- objetivo exacto;
- contexto de negocio;
- modo candado o límites;
- archivos/rutas a revisar;
- archivos/rutas prohibidos;
- tareas numeradas;
- validaciones;
- criterios de aceptación;
- formato final de reporte;
- instrucciones de no hacer commit/push si aplica.

Reglas:
- No inventes rutas si no las conocemos. Pregunta o usa placeholders claros.
- No pidas tocar secretos, credenciales, .env, pagos o deploy salvo que el
  usuario lo autorice expresamente.
- No mezcles fases.
- Si el trabajo es grande, divídelo en P0/P1/P2.
- Si falta contexto, prepara preguntas antes del prompt final.
- Si hay riesgo, dilo antes de proponer ejecución.

Formato preferido:
1. Preguntas pendientes si existen.
2. Prompt final listo para copiar.
3. Checklist de validación.
4. Riesgos que debe vigilar el usuario.

Regla principal:
Un buen prompt técnico no solo dice qué hacer. También dice qué no tocar, cómo
validar y cuándo detenerse.
```

### Primera prueba recomendada

```text
Quiero preparar un prompt para Claude Code/Codex.

Objetivo:
[explica la tarea]

Contexto:
[explica repo, producto o archivo si lo sabes]

Límites:
[di qué no se puede tocar]

Validaciones:
[di qué debería pasar al final]

Hazme preguntas si falta algo. Si no falta nada, crea el prompt final listo para
copiar.
```

## 9. Investigación inicial y plan de ataque

Antes de construir, investiga.

Tu proyecto de Estrategia y mejoras puede ayudarte a analizar:

- idea;
- proyecto;
- mercado;
- competencia;
- riesgos;
- demanda;
- oportunidades;
- qué no hacer todavía.

### Prompt de investigación inicial

```text
Quiero analizar esta idea antes de construir:
[describe la idea]

Haz una auditoría inicial con esta estructura:

1. Qué problema intenta resolver.
2. Para quién podría servir.
3. Qué evidencia tenemos.
4. Qué hipótesis estamos asumiendo.
5. Competidores o alternativas probables.
6. Riesgos de producto.
7. Riesgos de copy o promesa.
8. Riesgos técnicos.
9. Oportunidades.
10. Plan P0/P1/P2.
11. Qué NO construir todavía.
12. Próximo paso recomendado.

No quiero hype. Quiero una lectura fría y útil.
```

### Cómo interpretar la respuesta

No busques que la IA confirme tu idea. Busca que la mejore o la cuestione.

Una buena respuesta debe dejarte con:

- una decisión más clara;
- un P0 pequeño;
- riesgos visibles;
- menos cosas por hacer, no más.

## 10. Prompt 0

Prompt 0 es el prompt inicial para empezar cualquier trabajo técnico.

Sirve para que el proyecto IA prepare bien el encargo antes de llevarlo a Claude
Code, Codex o una herramienta similar.

No lo uses solo para código. También sirve para auditorías, documentación,
maquetación, QA o copy técnico si hay archivos y validaciones.

### Prompt 0 listo para usar

```text
Actúa como auditor técnico y Prompt Builder senior.

Necesito preparar un trabajo para Claude Code/Codex, pero antes quiero dejar
claro el objetivo, el contexto, los límites y las validaciones.

Objetivo:
[Qué quiero conseguir exactamente]

Contexto:
[Producto, repo, rama, página, archivo, negocio o situación]

Skills o capacidades necesarias:
[Frontend, backend, UX, copy, pagos, seguridad, documentación, etc.]

Repositorio o entorno:
[Ruta, rama, framework, restricciones conocidas]

Archivos o zonas a revisar:
[Lista si existe]

Archivos o zonas prohibidas:
[.env, secrets, pagos, deploy, media, APIs, etc.]

Límites:
[No hacer commit, no tocar X, no implementar P2, no borrar archivos, etc.]

Validaciones esperadas:
[build, typecheck, tests, diff check, búsqueda rg, QA visual, etc.]

Formato de reporte:
[Qué quiero que me devuelva al final]

Quiero que hagas lo siguiente:

1. Detecta si falta información crítica.
2. Si falta algo importante, pregúntame antes de crear el prompt.
3. Si hay información suficiente, crea un prompt final listo para pegar en
   Claude Code/Codex.
4. El prompt debe incluir objetivo, modo candado, tareas, validaciones,
   criterios de aceptación y formato final.
5. El prompt debe recordar que yo mantengo el control y que no se deben tocar
   archivos prohibidos.

No asumas permisos sensibles. No inventes rutas. No prometas resultados.
```

### Cuándo usar Prompt 0

Úsalo cuando:

- vas a tocar código;
- vas a pedir cambios en una web;
- vas a generar documentos entregables;
- vas a revisar checkout, pagos o emails;
- vas a pedir una auditoría técnica;
- vas a trabajar con archivos sensibles.

No hace falta usarlo para una pregunta simple.

## 11. Flujo candado

El flujo candado evita que pases de una idea a una ejecución sin control.

### Paso a paso

1. Proyecto IA audita y prepara prompt.
2. Usuario pega prompt en Claude Code/Codex.
3. Claude Code/Codex responde.
4. Usuario pega respuesta al proyecto IA.
5. Proyecto IA revisa.
6. Usuario decide siguiente paso.

> Tú eres el director de orquesta. La IA trabaja, pero tú lees, decides y
> mantienes el control.

### Por qué funciona

Funciona porque separa roles:

- el proyecto IA piensa, ordena y revisa;
- Claude Code/Codex ejecuta o propone cambios;
- tú decides y apruebas.

Cuando todo lo hace la misma conversación, es más fácil pasar por alto errores.
Cuando separas preparación, ejecución y revisión, reduces riesgo.

### Mini ejemplo

Quieres cambiar una landing.

Mal enfoque:

> "Hazme la landing mejor."

Flujo candado:

1. Proyecto IA analiza la landing, objetivo, claims prohibidos y validaciones.
2. Proyecto IA prepara un prompt con límites.
3. Claude Code/Codex implementa.
4. Pegas el resultado al proyecto IA.
5. Proyecto IA revisa si se cumplieron los límites.
6. Decides si se mergea, se corrige o se detiene.

## 12. Revisión de respuestas

No aceptes una respuesta técnica solo porque suena segura.

Revisa:

- qué hizo;
- qué tocó;
- qué no tocó;
- qué validó;
- qué no pudo validar;
- qué riesgos quedan;
- si hay errores;
- si hay que pedir otra fase.

### Checklist de revisión rápida

```text
1. Objetivo:
¿La respuesta resuelve el objetivo original?

2. Alcance:
¿Tocó solo lo permitido?

3. Archivos:
¿Qué archivos dice haber modificado o revisado?

4. Validaciones:
¿Qué comandos o pruebas ejecutó?

5. Riesgos:
¿Qué queda pendiente o no comprobado?

6. Contradicciones:
¿Hay algo que choque con las instrucciones iniciales?

7. Siguiente paso:
¿Tiene sentido avanzar, corregir o detenerse?
```

Si algo no cuadra, no sigas por inercia. Pide corrección o detén la fase.

## 13. Cierre profesional

Un trabajo con IA debe cerrarse de forma entendible.

Si no puedes explicar qué se hizo, qué se validó y qué queda pendiente, no tienes
un cierre profesional.

### Plantilla de informe final

```text
Objetivo:
[Qué se quería conseguir]

Cambios realizados:
[Qué se hizo]

Evidencias:
[Qué demuestra que funciona o que está preparado]

Validaciones:
[Build, typecheck, tests, revisión visual, búsquedas, etc.]

Riesgos pendientes:
[Qué falta revisar o comprobar]

Próximos pasos:
[Qué haría después]

No tocado:
[Archivos, sistemas o zonas sensibles que no se modificaron]
```

Usa esta plantilla incluso si trabajas solo. Te obliga a pensar con más claridad.

## 14. Plan de práctica recomendado

No intentes montar el sistema perfecto en una noche.

Hazlo así:

1. Crea contexto mínimo o recupera tu Pack Arranque IA.
2. Monta el proyecto Estrategia y mejoras.
3. Monta el proyecto Marketing y ventas.
4. Monta el proyecto Prompt Builder técnico.
5. Usa Prompt 0 para una tarea pequeña.
6. Ejecuta el flujo candado.
7. Revisa la respuesta.
8. Cierra con informe final.

Primera tarea recomendada:

> "Audita una página, idea, producto o documento y dime qué mejora P0 harías
> primero."

No empieces por un proyecto enorme. Empieza por algo pequeño, controlable y
revisable.
